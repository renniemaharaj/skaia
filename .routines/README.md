# routines

This directory contains routine definitions for project maintenance and automation. Each file describes a routine that models can perform. All routines must follow the strict template below.

---

## Routine File Template

```
# Routine: <routine_name>

## Purpose
<Brief description of the routine's purpose and scope.>

## Responsibilities
- <List of responsibilities and coverage areas>

## Goal
<Main objective of the routine>

## Principles
- <Guiding principles and rules>

## Workflow
1. <Step-by-step workflow for the routine>
2. ...

### Pseudocode
```

for item in scope:
perform routine steps
document findings

```

## Contexts
- <Relevant files, directories, or systems>

## Extension/See Also
- <Links to related routines, specs, or docs>

## Last Ran
- (To be updated by model)

## Last Run Remarks
- (To be updated by model)

## Next Steps
- (To be updated by model)
```

---

## Example Routines

- audit_backend — Backend auditing and validation
- audit_frontend — Frontend auditing and validation
- specs_specialist — Documentation and specs maintenance
- correctness — Consistency and anti-pattern checks

Add new routines as needed, always following the template above.
