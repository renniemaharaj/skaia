# Agent TODO System

This directory is used for collaborative planning and tracking of software engineering tasks by AI agents and developers. The files (no extension required) are used to:

- Plan and break down complex changes before implementation
- Iterate on migration, refactor, or feature plans across multiple models/agents
- Document decisions, strategies, and validation steps
- Ensure all models/agents understand the requirements and context before code changes

## Workflow

1. **Planning**: Add high-level plans, migration steps, or refactor strategies as entries in `.todo` files.
2. **Iteration**: Update, refine, or split plans as new requirements or insights emerge.
3. **Implementation**: Once a plan is finalized, a powerful agent or human can implement the steps.
4. **Validation**: Use `.todo` entries to track validation, testing, and review steps.

## Usage

- Each `.todo` file should focus on a specific area (e.g., migrations, frontend refactor, backend normalization).
- Use clear, actionable entries and update them as progress is made.
- Reference `.todo` entries in code comments or PRs to maintain traceability.

---

_This system helps ensure all agents and contributors are aligned and can collaborate effectively on complex software engineering tasks._
